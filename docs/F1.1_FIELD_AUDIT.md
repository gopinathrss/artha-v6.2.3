# F1.1 — Float → Decimal field audit

Audit of Prisma `Float` fields that represent money, units, NAV, FX, performance ratios, or other decimals that should use `Decimal` (Postgres `NUMERIC`). JSON columns holding money are listed but **not** migrated in Area 1 (per sprint spec).

Conventions:

| Kind | Proposed Prisma type |
|------|----------------------|
| Money (CZK, EUR, INR, USD headline amounts) | `Decimal @db.Decimal(14, 2)` |
| NAV / unit price | `Decimal @db.Decimal(14, 4)` |
| Units / share counts | `Decimal @db.Decimal(20, 8)` |
| FX rate (CZK per 1 unit quote) | `Decimal @db.Decimal(14, 6)` |
| Percent 0–100 (TER, allocation targets, gainPct) | `Decimal @db.Decimal(7, 4)` |
| Percent 0–1 or small ratios | `Decimal @db.Decimal(8, 6)` |
| Return / volatility / Sharpe / drawdown (performance) | `Decimal @db.Decimal(10, 6)` |

## Not Float (no change)

- AlertLog, MonthlyLetter, AIMemory, GeneratedReport: Json blobs only (no scalar Float money in schema).
- Int / Boolean / String / DateTime fields.
